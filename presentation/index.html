// Scheduler-first. If congestion makes it miss the block,
// *anyone* can fire an overdue will and collect a bounty
// that scales with lateness — capped at one flat fee.
pub fn trigger(origin, id: WillId) -> DispatchResult {
    let who  = ensure_signed(origin)?;
    let will = Wills::<T>::get(id).ok_or(Error::WillNotFound)?;
    ensure!(will.active(), Error::AlreadyExecuted);
    ensure!(now > will.expiry_block, Error::TooEarlyToTrigger);
    let reward = TriggerRewardPerBlock::get()
        .saturating_mul(blocks_overdue.into())
        .min(TriggerRewardCap::get());
    Self::do_execute(id, will, Some((who, reward)))
}

Fees split 30/70 — burn and treasury — via SplitTwoWays. Heartbeats are feeless. And the manual trigger rewards its caller, scaling with block overdue, so scheduler delays get cleaned up fast.

// Build the XCM that runs on Asset Hub as the owner.
fn build_ah_proxy_xcm(owner: &AccountId, inner: Vec<u8>) -> Xcm<()> {
    let fees: Asset = (Location::parent(), 100 * ROC).into();
    let proxy_call = ah_wrap_proxy(owner, inner);  // Proxy.proxy(owner, _, call)
    Xcm(vec![
        WithdrawAsset(fees.clone().into()),
        BuyExecution { fees, weight_limit: WeightLimit::Unlimited },
        Transact {
            origin_kind: OriginKind::SovereignAccount,
            call: proxy_call.into(),
            fallback_max_weight: None,
        },
    ])
}
[by 3:15] The Bequest enum lists every action we can dispatch on Asset Hub — transfer, sweep, proxy, multisig. When a will fires, we build an XCM Transact carrying a Proxy.proxy wrap, so Asset Hub dispatches as the owner. Estate Protocol never touches the assets.
